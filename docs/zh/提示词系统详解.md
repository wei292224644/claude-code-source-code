# Claude Code 提示词系统详解

## 概述

Claude Code 的提示词系统是一个多层次、多用途的架构，涵盖了从核心系统提示词到具体工具描述的各个方面。这些提示词共同定义了 Claude Code 的行为模式、工具使用方式、记忆管理和上下文处理策略。

---

## 一、提示词类型与位置

### 1. 核心系统提示词

**文件位置**: `src/constants/prompts.ts` 和 `src/buddy/prompt.ts`

这是整个提示词系统的基础，定义了 Claude Code 的核心行为准则。

**主要组件**:

- **Session Memory 模板** (`DEFAULT_SESSION_MEMORY_TEMPLATE`): 定义会话笔记的结构，包含 Current State、Task specification、Files and Functions、Workflow、Errors & Corrections 等section
- **Session Memory 更新提示词**: 指导如何在每次交互后更新会话笔记
- **Compact/Summarization 提示词**: 当上下文接近限制时，用于压缩对话历史

**内容示例**:
```typescript
// Session Memory 模板结构
export const DEFAULT_SESSION_MEMORY_TEMPLATE = `
# Session Title
_A short and distinctive 5-10 word descriptive title..._

# Current State
_What is actively being worked on right now? Pending tasks..._

# Task specification
_What did the user ask to build? Any design decisions..._

# Files and Functions
_What are the important files? In short, what do they contain..._
...
`

// Compact 摘要提示词
const BASE_COMPACT_PROMPT = `Your task is to create a detailed summary of the conversation so far...
Your summary should include the following sections:
1. Primary Request and Intent
2. Key Technical Concepts
3. Files and Code Sections
4. Errors and fixes
...
```

### 2. 系统提示词构建

**文件位置**: `src/constants/systemPromptSections.ts`

提供系统提示词的分段管理和缓存机制。

**核心概念**:

- `systemPromptSection()`: 创建一个可缓存的提示词片段
- `DANGEROUS_uncachedSystemPromptSection()`: 创建一个每轮都重新计算的提示词片段
- `resolveSystemPromptSections()`: 异步解析所有片段并返回字符串数组

**使用方式**:
```typescript
const dynamicSections = [
  systemPromptSection('session_guidance', () =>
    getSessionSpecificGuidanceSection(enabledTools, skillToolCommands),
  ),
  systemPromptSection('memory', () => loadMemoryPrompt()),
  systemPromptSection('language', () =>
    getLanguageSection(settings.language),
  ),
  // ... 更多片段
]
```

### 3. Buddy 系统提示词

**文件位置**: `src/buddy/prompt.ts`

负责构建完整的系统提示词，是实际发送给模型的主要提示词来源。

**核心函数**: `getSystemPrompt()`

**包含的提示词片段**:
1. **Intro Section**: "You are an interactive agent that helps users..."
2. **System Section**: 工具执行、权限、hooks 等基础说明
3. **Doing Tasks Section**: 执行任务的核心准则
4. **Actions Section**: 谨慎执行行动的指南
5. **Using Your Tools Section**: 工具使用偏好指导
6. **Tone and Style Section**: 语气和风格指南
7. **Output Efficiency Section**: 输出效率指南
8. **Dynamic Sections**: MCP 指令、语言偏好、输出样式、记忆等动态内容

**关键设计 - 动态边界**:
```typescript
export const SYSTEM_PROMPT_DYNAMIC_BOUNDARY = '__SYSTEM_PROMPT_DOUNDARY__'
```

这个边界标记将系统提示词分为两部分:
- **静态部分** (边界之前): 可缓存，适合所有用户共享
- **动态部分** (边界之后): 用户/会话特定，不可缓存

### 6. Compact 压缩提示词

**文件位置**: `src/services/compact/prompt.ts`

用于上下文压缩时生成对话摘要。

**核心函数**:
- `getCompactPrompt()`: 生成完整压缩提示词
- `getPartialCompactPrompt()`: 生成部分压缩提示词
- `formatCompactSummary()`: 格式化摘要输出
- `getCompactUserSummaryMessage()`: 生成用户可见的摘要消息

**三种压缩模式**:
1. **BASE**: 完整摘要所有对话
2. **PARTIAL**: 仅摘要最近消息
3. **UP_TO**: 摘要到指定点

## 二、注入方式

### 1. 系统提示词的注入

**入口**: `src/buddy/prompt.ts` → `getSystemPrompt()`

**流程**:

1. **构建工具列表**: `getTools()` 从 `src/tools.ts` 获取所有启用的工具
2. **获取工具描述**: 每个工具的 `getPrompt()` 方法返回其描述字符串
3. **组装系统提示词**: `getSystemPrompt()` 将所有片段组合成字符串数组
4. **发送到 API**: 通过 `src/services/api/claude.ts` 发送到 Claude API

**代码流程**:
```typescript
export async function getSystemPrompt(
  tools: Tools,
  model: string,
  additionalWorkingDirectories?: string[],
  mcpClients?: MCPServerConnection[],
): Promise<string[]> {
  // 1. 获取技能命令
  const [skillToolCommands, outputStyleConfig, envInfo] = await Promise.all([
    getSkillToolCommands(cwd),
    getOutputStyleConfig(),
    computeSimpleEnvInfo(model, additionalWorkingDirectories),
  ])

  // 2. 构建动态片段
  const dynamicSections = [
    systemPromptSection('session_guidance', () => ...),
    systemPromptSection('memory', () => loadMemoryPrompt()),
    systemPromptSection('language', () => ...),
    // ...
  ]

  // 3. 解析片段
  const resolvedDynamicSections = await resolveSystemPromptSections(dynamicSections)

  // 4. 返回提示词数组
  return [
    getSimpleIntroSection(outputStyleConfig),
    getSimpleSystemSection(),
    // ... 静态部分
    ...(shouldUseGlobalCacheScope() ? [SYSTEM_PROMPT_DYNAMIC_BOUNDARY] : []),
    // ... 动态部分
    ...resolvedDynamicSections,
  ].filter(s => s !== null)
}
```

### 2. 工具描述的注入

**方式**: 工具描述作为系统提示词的一部分，通过 `tools` 参数传递

**流程**:

1. 每个工具定义在 `src/tools/{ToolName}/{ToolName}.ts` 中
2. 工具定义包含 `getPrompt()` 方法或 `DESCRIPTION` 常量
3. 工具列表通过 `src/tools.ts` 的 `getAllBaseTools()` 收集
4. 最终通过 API 的 `tools` 参数发送

**示例 - AgentTool**:
```typescript
// src/tools/AgentTool/AgentTool.ts
export const AgentTool = buildTool({
  name: AGENT_TOOL_NAME,
  description: await getPrompt(agentDefinitions), // 动态获取提示词
  // ...
})
```

---

### 3. 记忆提示词的注入

**方式**: 记忆提示词通过专门的 Agent Fork 注入

**流程**:

1. **检测触发条件**: 当对话消息数达到阈值时触发
2. **创建 Fork**: 使用 `runForkedAgent()` 创建后台 Agent
3. **注入提示词**: 将 `buildExtractAutoOnlyPrompt()` 或 `buildExtractCombinedPrompt()` 作为系统提示词
4. **执行记忆写入**: Fork Agent 使用 FileWriteTool/FileEditTool 写入记忆文件

---

### 4. Compact 压缩提示词的注入

**方式**: 通过 API 消息发送

**流程**:

1. **检测上下文限制**: 当上下文接近 Token 限制时
2. **生成压缩消息**: 调用 `getCompactPrompt()` 或 `getPartialCompactPrompt()`
3. **发送到 API**: 将压缩提示词作为用户消息发送
4. **接收摘要**: API 返回压缩后的摘要
5. **格式化摘要**: 调用 `formatCompactSummary()` 处理结果

## 四、关键设计模式

### 1. 缓存优化设计

**静态/动态分离**: `SYSTEM_PROMPT_DYNAMIC_BOUNDARY` 标记之前的部分可全局缓存

```typescript
return [
  // 静态部分 - 可缓存
  getSimpleIntroSection(outputStyleConfig),
  getSimpleSystemSection(),
  // ...
  ...(shouldUseGlobalCacheScope() ? [SYSTEM_PROMPT_DYNAMIC_BOUNDARY] : []),
  // 动态部分 - 不可缓存
  ...resolvedDynamicSections,
]
```

**API 缓存实现**: 通过 `buildSystemPromptBlocks()` 将 system prompt 转换为带有 `cacheScope` 的结构化格式：

```typescript
// api.ts:splitSysPromptPrefix() 的处理逻辑
{
  system: [
    { text: "You are Claude Code...\n\n# System...", cacheScope: "global" },  // API 缓存
    { text: "Your memory...\n\nSession guidance...", cacheScope: null },    // 每次发送
  ]
}
```

| cacheScope 值 | 含义 | 使用场景 |
|--------------|------|---------|
| `"global"` | Anthropic API 全局缓存，多请求间复用 | 静态内容（身份定义、行为规则） |
| `"org"` | 组织级别缓存 | 未启用 global 时的回退 |
| `null` | 不缓存，每次都发送完整内容 | 动态内容（memory、session_guidance） |

**缓存条件**:
```typescript
// betas.ts:shouldUseGlobalCacheScope()
export function shouldUseGlobalCacheScope(): boolean {
  return (
    getAPIProvider() === 'firstParty' &&  // 仅 Anthropic 第一方 API
    !isEnvTruthy(process.env.CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS)
  )
}
```

> 注意: `cacheScope` 是 **Anthropic Messages API 的原生参数**，Claude Code 通过它告诉 API 哪些内容可以跨请求缓存复用，而不是 Claude Code 自己在本地实现缓存。

### 2. 提示词片段化

使用 `systemPromptSection()` 将提示词分解为可管理的单元:

```typescript
const dynamicSections = [
  systemPromptSection('session_guidance', () => getSessionSpecificGuidanceSection(...)),
  systemPromptSection('memory', () => loadMemoryPrompt()),
  systemPromptSection('ant_model_override', () => getAntModelOverrideSection()),
  // ...
]
```

### 3. Fork Agent 模式

对于后台任务（记忆提取、提示词建议），使用独立的 Fork Agent:

```typescript
const result = await runForkedAgent({
  promptMessages: [createUserMessage({ content: prompt })],
  cacheSafeParams,
  canUseTool,  // 限制工具使用
  skipTranscript: true,
  skipCacheWrite: true,
})
```

### 4. 模板与变量替换

支持 `{{variableName}}` 语法的模板:

```typescript
function substituteVariables(
  template: string,
  variables: Record<string, string>,
): string {
  return template.replace(/\{\{(\w+)\}\}/g, (match, key) =>
    Object.prototype.hasOwnProperty.call(variables, key)
      ? variables[key]!
      : match,
  )
}
```

## 五、总结

Claude Code 的提示词系统设计要点:

1. **分层架构**: 核心系统提示词 → 工具描述 → 专用提示词（记忆、压缩等）
2. **缓存优化**: 通过 `SYSTEM_PROMPT_DYNAMIC_BOUNDARY` 分离静态/动态内容
3. **片段化管理**: 使用 `systemPromptSection()` 统一管理提示词片段
4. **独立执行**: 后台任务通过 Fork Agent 隔离执行
5. **灵活组合**: 工具描述在 API 调用时合并到提示词数组
6. **模板支持**: 通过 `{{variable}}` 语法实现动态内容注入

这种设计使得提示词系统既能够提供一致的基础行为，又能够灵活适应不同的执行场景和用户需求。
